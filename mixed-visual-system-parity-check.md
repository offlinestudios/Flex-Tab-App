# Mixed Visual System Parity Check

**Date:** 2026-02-02  
**Benchmark:** Strava mobile landing page  
**Target:** FlexTab Landing Page (Mixed Visual System)  
**Objective:** Confirm perceived quality parity with Strava using UI screenshots (primary) + stock photography (secondary)

---

## Visual Hierarchy Compliance ✅

### Rule: Product UI motion = proof, Stock photography = context

**FlexTab Implementation:**
- ✅ Hero section: Banner photo (context) + UI screenshot (proof) - 50/50 split
- ✅ Social proof: Text-only with subtle detail shot background (5% opacity)
- ✅ Core value prop: Large UI screenshot (70%) + small stock photo (30%)
- ✅ Feature proof: 3 UI screenshots (primary) with no competing photography
- ✅ Final CTA: Text-focused with subtle detail shot background (5% opacity)

**Strava Comparison:**
- Strava hero: Cycling photo + app screenshot overlay (same 50/50 approach)
- Strava features: App screenshots dominate, photography provides context
- FlexTab matches Strava's visual hierarchy

**Verdict:** PARITY ACHIEVED

## Animation Restraint ✅

### Rule: Animations support comprehension, not decoration

**FlexTab Animations:**
- ✅ Hero load: 400ms fade + translate (only on initial load)
- ✅ Scroll: 400ms fade only (no translate, no bounce)
- ✅ Hover: 150ms scale(1.02) + shadow (subtle)
- ✅ Respects prefers-reduced-motion
- ✅ No count-up animations (would be distracting)

**Animation Purpose:**
- Fade-in: Reveals content progressively (supports comprehension)
- Hover: Provides feedback (supports interaction)
- No decorative motion

**Strava Comparison:**
- Strava has minimal motion on landing page
- FlexTab motion is equally restrained

**Verdict:** PARITY ACHIEVED
