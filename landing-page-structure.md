# FlexTab Landing Page Structure

**Reference Benchmark:** Strava mobile landing page design discipline  
**Domain:** Weightlifting / Strength Training  
**Tone:** Calm, serious, confident, non-promotional

---

## Section 1: Hero

**Purpose:** Establish immediate clarity and trust through restraint

**Psychological Goal:** Confidence through simplicity. No hype, just capability.

**Headline:** Track every rep. See every gain.

**Subhead:** The workout tracker built for progressive overload.

**CTA:** Start tracking free

**Content Density:** Very low

**Notes:**
- Single hero image showing clean UI screenshot or athlete in controlled environment
- Minimal text, maximum breathing room
- No marketing claims, just clear value proposition

---

## Section 2: Proof (Social Proof / Trust Signal)

**Purpose:** Establish credibility without testimonials

**Psychological Goal:** Trust through data, not opinion

**Headline:** Built for consistency

**Subhead:** Trusted by lifters who track every session.

**CTA:** None

**Content Density:** Very low

**Notes:**
- Simple stat display (e.g., "10,000+ workouts logged this week")
- No faces, no quotes, just numbers
- Subtle, not promotional

---

## Section 3: Feature Story (Primary Value Proposition)

**Purpose:** Explain the core mechanic in one clear story

**Psychological Goal:** Understanding before commitment

**Headline:** Progressive overload, made visible

**Subhead:** Every set logged. Every PR tracked. Every week compared.

**CTA:** None (scroll continues)

**Content Density:** Low

**Notes:**
- Single UI screenshot showing progression over time
- Focus on clarity: charts, numbers, trends
- No fluff, just the mechanism

---

## Section 4: Feature Grid (Secondary Capabilities)

**Purpose:** Show breadth without overwhelming

**Psychological Goal:** Competence through coverage

**Structure:** 3 feature cards, each with:
- Icon or small UI preview
- Feature name (2-3 words)
- One-sentence description

**Features:**
1. **Custom exercises** — Track any movement, any style.
2. **Body measurements** — Log weight, chest, arms, waist, thighs.
3. **Workout history** — Review every session, edit any set.

**Content Density:** Medium

**Notes:**
- Grid layout, equal weight to each
- No hierarchy within grid
- Clean, functional descriptions

---

## Section 5: CTA Block (Conversion)

**Purpose:** Clear, low-friction entry point

**Psychological Goal:** Momentum without pressure

**Headline:** Start your next session

**Subhead:** Free to use. No credit card required.

**CTA:** Get started

**Content Density:** Very low

**Notes:**
- Single button, center-aligned
- No secondary CTAs competing for attention
- Calm background (solid or subtle gradient)

---

## Section 6: Footer

**Purpose:** Legal and navigational closure

**Psychological Goal:** Professionalism and completeness

**Content:**
- Product name (FlexTab)
- Links: Privacy, Terms, Support
- Copyright notice

**Content Density:** Very low

**Notes:**
- Minimal, functional
- No social media icons unless essential
- Clean typography, subtle color

---

## Design Principles Applied

1. **Restraint over spectacle** — Every section earns its place
2. **Clarity over cleverness** — No metaphors, just mechanisms
3. **Data over opinion** — Show capability, not testimonials
4. **Breathing room over density** — White space is a feature
5. **Calm confidence** — No urgency, no hype, just quality

---

## Mobile-First Considerations

- Single-column layout throughout
- Generous vertical spacing between sections
- Touch-friendly CTA buttons (min 44px height)
- Readable font sizes (16px+ for body text)
- No horizontal scrolling
- Fast loading (optimized images, minimal animations)
